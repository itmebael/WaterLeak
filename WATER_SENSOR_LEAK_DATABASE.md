# Water Sensor Leak Detection - Database Storage

## Where Sensor Readings Are Saved

### 1. Regular Status Updates (Every 5 seconds)
**Table Name:** `water_connection_control`  
**Column:** `sensor_data` (JSONB field)

**IMPORTANT:** You need to add the `sensor_data` column to your table first!
Run the SQL in `add_sensor_data_column.sql` to add this column.

### 2. Leak Detections (When sensors >= 25%)
**Table Name:** `water_leak_detections`

### When Leaks Are Saved
Leaks are automatically saved to the database when:
- **Sensor 1** detects water (>= 25%) - Line 464 in `arduino_esp_water_control.ino`
- **Sensor 2** detects water (>= 25%) - Line 512 in `arduino_esp_water_control.ino`

Each sensor works **independently** - if Sensor 1 detects a leak, it saves immediately. If Sensor 2 detects a leak, it also saves immediately.

## Data Structure Saved

### Main Fields
```json
{
  "property_id": "9de4e6db-74ac-4b68-a0d1-290ba7fb50ae",
  "segment_id": "e6f043d2-f3ed-4ff4-b1d4-fb925434b7aa",
  "leak_type": "water_sensor",
  "severity": "medium" or "high",  // "high" if both sensors, "medium" if one sensor
  "status": "active",
  "location_description": "Kitchen - Water Sensor Detection",
  "estimated_water_loss_rate": 0.0,
  "flow_rate_anomaly": 0.0,  // Current flow rate
  "confidence_score": 0.95,
  "detection_date": "2024-01-15T22:16:16Z"  // Auto-generated by database
}
```

### Sensor Data (stored in `sensor_data` JSON field)
```json
{
  "sensor_data": {
    "water_sensor_1_value": 0,           // Raw ADC value (0-4095)
    "water_sensor_1_percent": 0.0,       // Percentage (0-100%)
    "water_sensor_1_detected": false,    // true if >= 25%
    "water_sensor_2_value": 883,         // Raw ADC value (0-4095)
    "water_sensor_2_percent": 98.9,      // Percentage (0-100%)
    "water_sensor_2_detected": true,     // true if >= 25%
    "flow_rate_lpm": 0.0,                // Current flow rate
    "valve_status": "closed",            // "open" or "closed"
    "detection_method": "water_sensor"
  }
}
```

## How to Query Leaks

### 1. View All Active Water Sensor Leaks
```sql
SELECT * FROM water_leak_detections
WHERE leak_type = 'water_sensor'
  AND status = 'active'
ORDER BY detection_date DESC;
```

### 2. View Leaks by Sensor
```sql
-- Sensor 1 leaks only
SELECT * FROM water_leak_detections
WHERE leak_type = 'water_sensor'
  AND (sensor_data->>'water_sensor_1_detected')::BOOLEAN = true
ORDER BY detection_date DESC;

-- Sensor 2 leaks only
SELECT * FROM water_leak_detections
WHERE leak_type = 'water_sensor'
  AND (sensor_data->>'water_sensor_2_detected')::BOOLEAN = true
ORDER BY detection_date DESC;

-- Both sensors detected
SELECT * FROM water_leak_detections
WHERE leak_type = 'water_sensor'
  AND (sensor_data->>'water_sensor_1_detected')::BOOLEAN = true
  AND (sensor_data->>'water_sensor_2_detected')::BOOLEAN = true
ORDER BY detection_date DESC;
```

### 3. View Sensor Percentages
```sql
SELECT 
  id,
  detection_date,
  (sensor_data->>'water_sensor_1_percent')::NUMERIC as sensor1_percent,
  (sensor_data->>'water_sensor_2_percent')::NUMERIC as sensor2_percent,
  (sensor_data->>'water_sensor_1_detected')::BOOLEAN as sensor1_detected,
  (sensor_data->>'water_sensor_2_detected')::BOOLEAN as sensor2_detected,
  severity,
  status
FROM water_leak_detections
WHERE leak_type = 'water_sensor'
ORDER BY detection_date DESC;
```

### 4. Recent Leaks (Last 24 Hours)
```sql
SELECT * FROM recent_water_sensor_leaks;
```

### 5. Using SQL Functions (from water_sensor_leak_save.sql)
```sql
-- Get leaks by sensor
SELECT * FROM get_leaks_by_sensor(1);  -- Sensor 1 only
SELECT * FROM get_leaks_by_sensor(2);  -- Sensor 2 only
SELECT * FROM get_leaks_by_sensor(3);  -- Both sensors

-- Get statistics
SELECT * FROM get_sensor_leak_statistics();
```

## API Endpoint

The ESP32 sends data to:
```
POST https://ituksombwexvutmxcmsv.supabase.co/rest/v1/water_leak_detections
```

With headers:
- `Content-Type: application/json`
- `apikey: [your-supabase-key]`
- `Prefer: return=representation`

## Resolving Leaks

When sensors return to dry state (< 25%), you can manually resolve leaks:

```sql
UPDATE water_leak_detections
SET 
  status = 'resolved',
  resolved_date = NOW(),
  resolution_notes = 'Sensors returned to dry state (<25%)'
WHERE id = 'leak-uuid-here';
```

Or use the SQL function:
```sql
SELECT resolve_water_sensor_leak('leak-uuid-here', 'Sensors dry');
```
